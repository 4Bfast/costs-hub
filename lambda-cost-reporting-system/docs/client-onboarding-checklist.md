# Client Onboarding Checklist

## Pre-Onboarding Preparation

### Client Information Gathering
- [ ] Company name and contact information
- [ ] Primary contact person and role
- [ ] AWS account IDs to be monitored
- [ ] Email addresses for report recipients
- [ ] Preferred report frequency (weekly/monthly)
- [ ] Cost threshold preferences
- [ ] Branding requirements (logo, colors)

### Technical Requirements
- [ ] AWS account access confirmed
- [ ] Cost Explorer enabled in client accounts
- [ ] IAM permissions discussed and planned
- [ ] Email domain verification requirements identified
- [ ] Security and compliance requirements reviewed

## AWS Setup (Client Side)

### IAM Configuration
- [ ] IAM user created for cost reporting
- [ ] Cost Explorer policy attached
- [ ] Access keys generated and secured
- [ ] Permissions tested with AWS CLI
- [ ] Cross-account role configured (if applicable)

### Account Verification
- [ ] AWS account IDs verified (12-digit format)
- [ ] Primary regions identified
- [ ] Multi-account setup documented (if applicable)
- [ ] Cost data availability confirmed

## System Configuration

### Client Configuration File
- [ ] JSON configuration file created
- [ ] Client information populated
- [ ] AWS account details added
- [ ] Email recipients configured
- [ ] Report preferences set
- [ ] Branding configuration added
- [ ] Configuration file validated

### CLI Operations
- [ ] Sample configuration generated
- [ ] Client configuration validated
- [ ] AWS access tested
- [ ] Client added to system
- [ ] Client status verified as active

## Asset Management

### Branding Assets
- [ ] Company logo received and processed
- [ ] Logo uploaded to S3 bucket
- [ ] Logo S3 key updated in configuration
- [ ] Brand colors validated (hex format)
- [ ] Email footer text configured

### Testing
- [ ] Test report generation executed
- [ ] Branding applied correctly
- [ ] Email template rendering verified
- [ ] Asset accessibility confirmed

## Email Configuration

### SES Setup
- [ ] Sender email address verified
- [ ] Recipient email addresses verified (if in sandbox)
- [ ] Email templates tested
- [ ] Delivery testing completed
- [ ] Bounce/complaint handling configured

### Delivery Verification
- [ ] Test emails sent successfully
- [ ] Email formatting verified
- [ ] Spam filter considerations addressed
- [ ] Client email receipt confirmed

## Validation and Testing

### Configuration Validation
- [ ] Client configuration syntax validated
- [ ] AWS permissions verified
- [ ] Email addresses format checked
- [ ] Threshold values validated
- [ ] Multi-account setup tested (if applicable)

### End-to-End Testing
- [ ] Manual report generation tested
- [ ] Cost data retrieval verified
- [ ] Report formatting confirmed
- [ ] Email delivery successful
- [ ] Branding application verified

## Documentation and Communication

### Internal Documentation
- [ ] Client details recorded in system
- [ ] Special requirements documented
- [ ] Configuration backup created
- [ ] Onboarding notes completed
- [ ] Support contacts identified

### Client Communication
- [ ] Onboarding completion confirmed
- [ ] Report schedule communicated
- [ ] Sample report provided
- [ ] Support contact information shared
- [ ] Feedback process explained

## Post-Onboarding Monitoring

### Initial Monitoring
- [ ] First scheduled report execution monitored
- [ ] Email delivery confirmed
- [ ] Client feedback collected
- [ ] Any issues identified and resolved
- [ ] Success metrics recorded

### Ongoing Setup
- [ ] Monitoring alerts configured for client
- [ ] Regular review schedule established
- [ ] Maintenance procedures documented
- [ ] Escalation procedures defined
- [ ] Client satisfaction confirmed

## Quality Assurance

### Technical Verification
- [ ] All AWS accounts accessible
- [ ] Cost data complete and accurate
- [ ] Report generation working correctly
- [ ] Email delivery reliable
- [ ] Error handling tested

### Business Verification
- [ ] Client requirements met
- [ ] Report content satisfactory
- [ ] Branding applied correctly
- [ ] Schedule meets expectations
- [ ] Support process understood

## Sign-off

### Technical Sign-off
- [ ] System administrator approval
- [ ] Technical testing completed
- [ ] Configuration validated
- [ ] Monitoring configured
- [ ] Documentation complete

### Business Sign-off
- [ ] Client satisfaction confirmed
- [ ] Requirements met
- [ ] Service level agreements established
- [ ] Support procedures communicated
- [ ] Onboarding officially complete
